# CLAUDE.md — SafeRedact

## プロジェクト概要

**SafeRedact** は、テキストをAIに渡す前に個人情報（氏名・住所など）をマスキングし、AI処理後に元に戻すためのツールです。ビルドシステムやサーバー不要の、シングルファイルHTMLアプリです。

**基本的な使い方：**
1. AES-256暗号化された辞書ファイル（.json）を開くか新規作成する
2. テキストを入力欄に貼り付け、マスキングルールを適用して安全なテキストを生成する
3. マスキング済みテキストをAIに渡す
4. AIの返答を復元欄に貼り付け、元の名前に戻す

## ファイル構成

## アプリの起動方法

`index.html` をブラウザ（Chrome 86以降、Edge 86以降、Firefox 111以降、Safari 15.2以降）で開くだけです。

```bash
# ローカルサーバーを立てる場合（File System Access API の動作が安定する）：
python3 -m http.server 8080
# その後 http://localhost:8080 を開く
```

**注意：** File System Access API はHTTPSまたはlocalhostでのみ動作します。`file://` で直接開くと、ブラウザによってはファイル保存機能が使えないことがあります。

### 暗号化の仕様

- アルゴリズム：AES-256-GCM
- 鍵導出：PBKDF2、SHA-256、200,000回、ランダム16バイトのsalt
- IV：暗号化ごとにランダム12バイトを生成
- 保存形式：`salt(16) + iv(12) + 暗号文` → Base64 → JSONフィールド

辞書ファイルはディスク上では常に暗号化されています。平文がファイルシステムに書き込まれることはありません。

## コーディング規約

### 命名
- JS識別子はすべて **camelCase**
- 要素IDは動詞プレフィックス付き：`btnRunAll`、`inputText`、`dictTableBody` など
- UI文字列とコメントは **日本語**

### パターン
- **フレームワークなし** — `getElementById`・`querySelector`・イベントリスナーで直接DOM操作
- **変更時に全再描画** — 辞書の状態が変わるたびに `renderDict()` でテーブル全体を再構築
- **モーダル** — `.show` CSSクラスのオン・オフで表示切替（アニメーションライブラリなし）
- **非同期処理** — File System Access API・Web Crypto API はすべて `async/await` で記述
- **防御的ガード** — `dictFileHandle` や `dictPassword` が未設定の場合は関数の先頭で早期リターン

### 自動保存
辞書を変更するたびに `saveDict()` が呼ばれ、暗号化してファイルに書き戻します。成功すると「自動保存」が一瞬表示されます。

## 辞書ファイルの形式（.json）

ディスク上のファイルは以下のJSON構造です：

```json
{
  "v": 1,
  "data": "<Base64エンコードされたAES-256-GCM暗号文>"
}
```

復号後の `data` はJSON配列です：

```json
[
  { "original": "田中太郎", "replacement": "[利用者A]", "date": "2026-04-04" },
  ...
]
```

エクスポートのバックアップファイル名は `SR-YYYYMMDD.json` 形式です。

1. `index.html` を直接編集する（コンパイル不要）
2. ブラウザをリロードして動作確認
3. デバッグはブラウザの開発者ツール（コンソール）で。`dict`・`dictPassword` 等はグローバル変数として参照可能
4. 変更をコミットしてpush

```bash
git add index.html

```

## テスト方法

自動テストはありません。手動で以下を確認します：

- `index.html` を Chrome/Edge で開く
- テスト用パスワードで辞書ファイルを新規作成
- いくつかのルールを追加（元の語 → 置換語）
- サンプルテキストに「全置換」を適用
- 復元パネルで逆置換を確認
- 暗号化バックアップをエクスポートし、再インポート（マージ・置換の両方）
- 辞書を変更するたびに自動保存が動作することを確認

## 制約事項

- **File System Access API** — Firefox（v110以前）や `file://` URLでは使えない場合があります。その場合はダウンロード・アップロードによる代替手段が表示されます。
- **シングルファイル** — すべて `index.html` に収めること。明示的な判断なしにビルドシステム導入や複数ファイル分割はしないこと。
- **日本語UI** — 多言語対応を追加しない限り、ユーザー向け文字列は日本語のままにすること。
- **ネットワーク不使用** — オフラインで動作すること。外部サービスへのfetch/XHRを追加しないこと。
- **`.gitignore` が `*.json` を除外** — 辞書バックアップファイルはバージョン管理に含めないこと。

## セキュリティ上の注意

- `dictPassword` をコンソールやエラーメッセージに出力・露出させないこと。
- セッション中、`dict` 配列はメモリ上に平文の個人情報を保持します。`localStorage` や `sessionStorage` に平文で書き出す機能を追加しないこと。
- PBKDF2のイテレーション回数は `200000`。この値を下げないこと。
- 暗号化のたびに新しいランダムsaltとIVを生成すること。使い回しは禁止。
